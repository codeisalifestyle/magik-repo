# Product names: short, evocative, no functional descriptors

## Decision

New SKUs will be named with **single-word, evocative, non-descriptive names** (e.g., "Halcyon," "Ember," "Sable"). Functional descriptors ("Sleep Aid," "Daily Multi") go in the *subtitle*, never the product name itself.

## Rationale

- The product portfolio will grow into adjacent benefits over time. Functional names lock us into a single use case and become liabilities when we expand.
- Evocative names are searchable as standalone terms (low-competition SEO) and trademarkable far more easily than descriptive ones.
- Customer interviews showed "feel" associations stick longer than "function" associations for repeat purchase.

## Counter-examples

- "Lumera Sleep" — rejected. Locks the SKU to one benefit; competes with "AG1 Sleep" / "Olly Sleep" type SEO; not trademarkable as a stand-alone.
- "DeepRest" — rejected. Mashed-up compound, common in the category; reads as generic.

## Process

When proposing a new SKU name:
1. Check the existing portfolio for thematic overlap.
2. Verify USPTO and `.com` availability.
3. Run a 5-person customer-panel association test before committing.
